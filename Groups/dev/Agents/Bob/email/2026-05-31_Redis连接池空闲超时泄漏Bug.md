## Bug 概要

| 项目 | 内容 |
|------|------|
| **标题** | Redis 连接池偶尔在空闲超时时无法正确关闭连接 |
| **严重程度** | P1 |
| **发现时间** | 2026-05-31 |
| **关联 PR** | 上次 Redis 连接池修复 PR（Dispose 修复 + timeout 处理） |

## 复现步骤

1. 启动应用，正常进行 Redis 读写操作（连接池正常工作）
2. 停止所有 Redis 操作，让连接进入空闲状态
3. 等待约 **30 分钟**（触发 idle timeout 阈值）
4. 再次发起 Redis 请求
5. 观察：旧连接未被正确 Dispose，新连接被创建，但旧连接仍占据资源

## 预期行为 vs 实际行为

- **预期**: idle timeout 触发后，连接池正确关闭空闲连接并释放资源
- **实际**: 部分空闲连接未被关闭，导致连接句柄泄漏，长时间运行后连接池耗尽

## 怀疑方向

1. `Dispose()` 方法中 idle timeout 回调的条件判断可能有 race condition
2. 超时回调和主动关闭之间可能存在状态竞争
3. 连接状态标记（如 `_disposed` / `_closed`）在 idle timeout 路径下未正确设置

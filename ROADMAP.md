# Mind Agency Roadmap

## 竞品对比：Mind Agency vs Multica

### 我们的优势（核心竞争力）

| 功能 | Mind Agency | Multica |
|------|:-----------:|:-------:|
| 共识投票 + 对抗性审查 + 多轮辩论 | ✅ | ❌ |
| 三层记忆 + 语义搜索 | ✅ | ❌ |
| Agent 间邮件协议 | ✅ | ❌ |
| 信号驱动自动响应 + 防抖 | ✅ | ❌ |
| 工作流断点恢复 + 崩溃恢复 | ✅ | ❌ |
| DLQ + Outbox 模式 | ✅ | ❌ |
| 审计日志 | ✅ | ❌ |
| 可视化 DAG 编辑器 | ✅ | ❌ |
| 人工审批节点 | ✅ | ❌ |
| 动态 YAML 热重载 | ✅ | ❌ |
| Agent 行为画像 | ✅ | ❌ |
| 本地嵌入（无模型下载） | ✅ | ❌ |

### 我们缺的功能

| 功能 | 优先级 | 状态 | 说明 |
|------|:------:|:----:|------|
| 多工作空间隔离 | P0 | ⬜ | 没这个没法服务多团队，最高优先级 |
| CLI 工具 | P0 | ⬜ | 开发者主要入口，`mind agent/group/workflow` |
| Issue 追踪 + Board 视图 | P1 | ⬜ | 团队协作刚需，结构化任务管理 |
| Skills 技能库 | P1 | ⬜ | 工作流复用，经验沉淀 |
| Docker 自托管部署 | P1 | ⬜ | Dockerfile + docker-compose.yml |
| 认证系统 | P1 | ⬜ | 多人使用必须 |
| 多供应商支持 | P2 | ⬜ | 目前只有 claude + codex，需扩展 |
| Webhook 接入 | P2 | ⬜ | 外部系统触发工作流 |
| 移动端 | P3 | ⬜ | 可后加 |
| PostgreSQL 后端 | P3 | ⬜ | 文件系统已够用，大 scale 再考虑 |

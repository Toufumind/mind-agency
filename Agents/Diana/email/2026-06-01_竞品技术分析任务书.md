## 三、分析维度（每个竞品逐项对比）

1. **消息 / Event Bus 架构** — 与我们的 Event Bus v0.2 对比：性能、类型安全、通配符路由、持久化
2. **Agent 调度机制** — 后台轮询 vs 事件驱动 vs 长连接；与我们的 30s Scheduler v1 对比
3. **Agent 通信协议** — 群聊 vs 点对点 vs 广播；消息格式、@提及、去重
4. **自治程度** — 是否需要人工触发？能否自我组织 Sprint？
5. **多 Agent 并发模型** — 并行轮询、worktree 隔离、token 预算
6. **可扩展性** — 新增 Agent 的成本、群组机制、权限模型
7. **工程成熟度** — TypeScript 类型安全、测试覆盖、文档质量
